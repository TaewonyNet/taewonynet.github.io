### 1. 서론 (Introduction)
- **문제/상황 (Problem):**
    - 프로젝트 또는 업무상 알아야 할 용어가 너무 많다. 
- **목적 (Purpose):**
    - Gemini를 활용해 용어사전을 자동으로 만들어 향 후 사용할 수 있도록 한다.
- **대상 (Target Audience):**
    - 맨날 잊어버리는 나같은 사람을 위해.

### 2. 방법 및 과정 (Methods & Process)

- **배경 조사 및 데이터 (Data Collection):**
    - 용어 사전을 만들기 위한 간단한 템플릿을 조사한다.
    - 복잡한 템플릿은 비용(?)이 많이 청구되므로 최대한 간결하게 작성한다.
- **접근 방법 (Approach Methods):**
    - **[방법 1]:** 입력 텍스트에서 용어를 추출하는 표준 프롬프트를 설계한다.
    - **[방법 2]:** Gemini API 호출로 결과를 JSON 형식으로 받을 수 있도록 조정한다.
- **분석 및 해결 프로세스 (Analysis Flow):**
    - **도구/기술:** Gemini API를 사용한다.
    - **주요 단계:** 프롬프트 템플릿 작성 $\rightarrow$ API 호출 $\rightarrow$ JSON 출력 저장.
    - **결과 도출 및 검증:** 용어 추출 정도를 확인하고 json형태가 유효한지 확인한다.

### 3. 결과 (Results)
- **분석 결과 요약:**

## 폴더를 기준으로 용어사전을 만들자
실제 용어사전을 만들어 향 후 사용 가능하게 해보자

### 1. 서론 (Introduction)

- **문제/상황 (Problem):**
    - 향 후 검색이나 RAG 등으로 활용 할 수 있는 파일기반 용어사전을 만들고싶다. 
- **목적 (Purpose):**
    - 폴더를 지정하면 문서를 자동 분석하여 용어사전을 생성한다.
- **대상 (Target Audience):**
    - 문서 정리하기 귀찬은 나같은 사람.

### 2. 방법 및 과정 (Methods & Process)

- **배경 조사 및 데이터 (Data Collection):**
    - 내부 파일이나 컨탠츠를 이용하여 용어사전을 만드는 방법 확인한다.
- **접근 방법 (Approach Methods):**
    - **[방법 1]:** 파일 목록을 획득하여 LLM에 표준화 프롬프트를 사용하여 Json 형태로 추출한다.
    - **[방법 2]:** 용어는 dict형태로 기록하여 기존 항목은 넣지 않으며 처리된 파일은 기록한다. 
- **분석 및 해결 프로세스 (Analysis Flow):**
    - **도구/기술:** Python, Gemini API.
    - **주요 단계:** Json로드 $\rightarrow$ 디렉토리에서 파일 획득 $\rightarrow$ API 호출과 결과 파싱 $\rightarrow$ 용어 병합 후 저장.
    - **결과 도출 및 검증:** JSON 파싱 성공과 용어 중복 여부를 확인하며 파일 처리 로그를 확인한다.

### 3. 결과 (Results)

- **분석 결과 요약:**
    - 폴더의 파일들이 정상 처리되어 용어사전 term.json에 용어들이 누적 저장된다.

### 4. 인사이트 및 액션 (Insights & Action)

- **인사이트 (Insight):**
    - 파일 기준 용어 사전으로 편하게 용어사전을 유지할 수 있다.
- **실행 방안 (Action Plan):**
    - 글 작성 후 용어사전을 실행 해 주기적으로 업데이트한다.
- **한 줄 결론 (Key Takeaway):**
    - 필요한 용어사전을 쉽과 간편하게 관리한다.
- **다음 스텝 (Next Step):**
    - 용어사전을 별도의 페이지로 구축하여 쉽게 검색 가능하도록 한다.
